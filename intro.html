HTML5 Canvas

Introduction

One of the coolest features introduced in HTML5 is the canvas element which allows you to draw directly onto a web page. You will be using JavaScript to do your drawing so I assume you have some basic knowledge of the language. By the end of this tutorial you will know the basic uses the canvas element has to offer and will be well on your way to creating some simple games.

Getting started with canvas

Using the canvas element in your project couldn't be easier. The following code is all you need to get started.

<canvas id="mainCanvas">
     <!-- insert fallback  -->
</canvas>

Now before you go and run this code let me first tell you that it does absolutely nothing on its own. What this bit of code gets you is a blank canvas element but once we get into the 2d rendering context you will start to see some results.

2d rendering context

The canvas element is pretty lame but, the 2d rendering context is awesome and where all the cool things happen. It is important that you understand that everything you draw gets drawn on the 2d rendering context not the canvas. The canvas is only used to access and display the rendering context.
